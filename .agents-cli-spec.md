# Agent Spec

This file is the `agents-cli` workflow's Phase 0 entry point. It intentionally
does not duplicate content: `spec/` is this repository's single source of
truth (see `.agents/AGENTS.md`), so each section below points at the
authoritative spec location instead of restating it.

## Overview

See `spec/technical_specification.md` §3.3 ("Context Assembly & the
Discussion Agent") and `spec/user_specification.md`. In short: a tool-using
discussion agent that helps a reader discuss passages of an uploaded
document — suggestions and journal generation are deliberately non-agentic
(user-spec §7) and out of scope for this agent.

## Example Use Cases

`spec/features/*.feature` (Gherkin scenarios) are the authoritative
behavioral examples.

## Tools Required

`spec/technical_specification.md` §3.3 and §8, and
`spec/contracts/agent-contract.yaml`:
- `search_document` — ephemeral, per-call in-memory SQLite FTS5 (BM25)
  keyword search scoped to the current workspace's document.
- `web_search` — ADK built-in Google Search grounding tool.
Both tools are read-only (`spec/features/security.feature`).

## Constraints & Safety Rules

`spec/features/security.feature` and the untrusted-content wrapping in
`spec/contracts/agent-contract.yaml`: all document text, notes, discussion
history, journal, and tool results are wrapped as delimited untrusted data;
the agent must not treat embedded instructions as commands; tools are
read-only and scoped server-side to the current workspace.

## Success Criteria

Deferred to §3.4 (eval assets) once agent logic exists — see

## Reference Samples

Checked `references/samples.md` (google-agents-cli-workflow skill):
`rag-agent-search` / `rag-vector-search` do not match — spec §9 explicitly
excludes RAG/embedding-based retrieval, and document search here is a
custom ephemeral FTS5/BM25 tool, not Discovery Engine or vector search.
`deep-search` does not match either — this agent's web-search tool is a
single ADK built-in grounding call, not iterative multi-agent research. No
sample applies; scaffolded from the base `adk` template instead.
